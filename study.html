1. What is General Systems Theory?

  • Provides a holistic perspective on designing and organizing complex systems.

2. Who came up with the idea of General Systems Theory?

  • Ludwig von Bertalanffy

3. What is Organizational Structure?

  • Defines how activities are directed toward achieving organizational aims.
  • Refers to how an organization manages resources like people, processes, and technology to support information systems development and management.
  • Plays a key role in designing and implementing effective information systems.

9. What is Software Testing?

  • The process of evaluating and verifying that a software product meets specified requirements.

10. What is Unit Testing?

  • A process where the smallest testable parts of an application (units) are individually tested for functionality.

11. What is White Box Testing?

  • A testing approach where the internal workings of the software, including code and integrations, are inspected.

12. What is Black Box Testing?

  • A testing technique focusing on software inputs and outputs without considering its internal design or code.

13. What is Integration Testing?

  • Testing where different units, modules, or components of an application are combined and evaluated as a single entity.

14. What is System Testing?

  • Evaluates how all components of an integrated application work together in a full system.

15. What is Regression Testing?

  • Testing conducted after code updates to ensure new logic doesn’t introduce defects in existing functionality.

16. What is Risk Management?

  • The process of identifying, assessing, and controlling risks to an organization’s operations and goals.

21. Draw the Waterfall SDLC Model.

22. List the Stages in the Waterfall Model:

  • Requirement Analysis/Feasibility Study
  • Design
  • Implementation
  • Testing
  • Deployment
  • Maintenance

23. What Happens in Each Stage of the Waterfall Model?

  • Requirement Analysis: Gather and document system requirements.
  • System Design: Create a detailed design based on requirements; no changes allowed afterward.
  • Implementation: Build the system as per the design in a step-by-step process.
  • Testing: Evaluate the product for bugs and confirm functionality.
  • Maintenance: Release the product to customers and address any issues or add new features.

24. What is an Advantage of the Waterfall Model?

  • Clear structure with defined goals and constant requirements throughout the lifecycle.

25. What is a Disadvantage of the Waterfall Model?

  • Does not easily accommodate changes during a project.
